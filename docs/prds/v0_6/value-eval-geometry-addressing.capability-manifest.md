## Field-population branch (Value::Undef sentinel)

R2a/R2b/R3b each replace a `Value::Undef` production with a real value on the eval path — the manifest's anti-`Undef` sentinel is **inverted** here: these leaves are *eliminating* silent `Undef`, and Rung 1 (R1a) is the floor that makes any *residual* `Undef`-from-non-realization **loud** rather than silent. A leaf PASSES field-population iff its producer writes a real `Value::Selector`/symbolic `Value::GeometryHandle`/finite Φ-projected `Value::Scalar` on the production path (`geometry_ops.rs`, `modal_ops.rs`), and FAILS if the only construction lives under `tests/`. The two-way boundary tests (PRD §7) are the field-population proof: symbolic-handle `content_hash` == realized-handle `content_hash`; eval-resolved node-set == kernel-resolved node-set.

## Numeric-floor branch (G6)

- **R3b/R3c** assert a Φ-projected transient response. **Do NOT assert float-equality on the response value** — modal numerics (mode truncation, Duhamel `O((ΩΔt)²)`, mesh density) make a tuned bound a false premise (cf. overlay G6 hazards esc-3821-44 Duhamel, esc-3453 bending lock). The numeric floor is **verdict-shaped**: assert (a) the **correct node** is selected (a non-antinode location selects a *different* node than the antinode fallback — the capability flip), (b) the response is **finite and ≥ 0** (no `Undef`), and (c) eval-resolved node-set **== kernel-resolved node-set** (two-way parity). Node-selection correctness and parity are exact; the response magnitude is bounded only as finite/non-degenerate. `bound > floor` holds because no magnitude bound is asserted.
- **R1a/R2a/R2b** assert no numeric bound (diagnostic emission, symbolic value presence, content-hash identity) → numeric-floor N/A.

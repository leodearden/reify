## Anti-orphan / anti-inversion summary

- **No orphan producer:** every tool is consumed by an e2e scenario (its own leaf signal) and again by the `E1` consumer suite; `layoutStore` (L0) is consumed by `resize_panes` (C2 deps L0); the testid/expose sweep (T0) is consumed by `C1`/`C2`. No tool lands without a real-GUI caller. The two hub files (`debug_server.rs`, `bridge.ts`) host the dispatch arms + handlers — the existing seam, not a new one.
- **No field-population inversion:** `get_diagnostics` (R2) reads the real production-path `compileDiagnostics` field, not a `tests/` construction. `inject_diagnostics` (F1) is the one synthetic input, **labelled**, with its signal pinned to the real UI render — so a silently-store-only landing fails its own gate.
- **No grammar fiction:** no novel `.ri` syntax; fixtures are ordinary `.ri`. The one capability grant (`core:window:allow-set-size`) and the one store lift (`layoutStore`) are confirmed-real substrate edits, owned in-task.
- **No false premise:** the three risky claims — "synthetic == real input," "`ui_outline` == AX tree," "signals are CI-gated" — are each **explicitly negated** in the PRD §3/§G6 and bounded to what the substrate actually supports. Coordinates carry no numeric floor (exact px + reported DPR). Real-input injection and true AX auditing are deferred (§5, AX‑1), not asserted.

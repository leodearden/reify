## Phase 2: Architectural Coherence

### High-Severity Findings

**S1 — Extrude/revolve not wired in compiler (unintended stub)**
Tasks 308/309 marked done, eval layer and OCCT FFI fully handle `SweepKind::Extrude`/`Revolve`, but `is_geometry_function()` and `compile_geometry_call()` have no entries for "extrude"/"revolve". User code `extrude(...)` produces a diagnostic error and no geometry.
*Location:* `crates/reify-compiler/src/lib.rs:402, 5036`
*Task:* #489

**S2 — Sub/Minimize/Maximize silently dropped in guarded blocks**
`compile_guarded_members()` has an empty `_ => {}` catch-all that silently discards Sub, Minimize, and Maximize declarations inside `where{}` blocks. No diagnostic emitted.
*Location:* `crates/reify-compiler/src/lib.rs:4376-4379`
*Task:* #490

**CP3/DR2 — Cross-template auto-param confusion in incremental paths**
`eval()` correctly resolves auto params per-template. `edit_param` and `resolve_concurrent_edit` collect auto params from ALL templates via HashMap iteration into a single solver call. `scope_name` is picked non-deterministically — wrong optimization objective may be applied.
*Location:* `crates/reify-eval/src/lib.rs:681-699 vs 1264-1392`
*Task:* #491

**CP6 — Second guard re-elaboration overwrites solver values**
`edit_param` has two guard re-elaboration blocks with the solver running between them. If an auto param lives in a guarded group whose guard becomes false, Block B (post-solver) overwrites the solver's resolved value with Undef.
*Location:* `crates/reify-eval/src/lib.rs:1562-1646, 1655-1797, 1799-1881`
*Task:* #492

**DR1 — `reify build` exits 0 on violated constraints**
`cmd_build` returns `ExitCode::SUCCESS` whenever geometry is produced, even if constraints are violated. `cmd_check` correctly exits with FAILURE. CI cannot detect violated constraints from build.
*Location:* `crates/reify-cli/src/main.rs:173-180`
*Task:* #493
